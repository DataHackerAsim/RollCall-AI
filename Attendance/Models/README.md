# `Attendance/Models/` — per-student face embeddings

This directory holds one `.npy` file per enrolled student. Each file contains a
reference face embedding (512-D float32 vector, L2-normalised) generated by the
InsightFace ArcFace model.

**No embeddings are shipped with this repository.** Populate it in one of two ways:

1. **Face Manager UI** — upload reference photos in `/face-manager/` and use the
   *Cluster → Label → Retrain* workflow. The UI writes `<First_Last>_embeddings.npy`
   here automatically.

2. **Scripted enrollment** — call `pipeline.retrain_student_embedding(student_id)`
   after attaching sample crops to a `FaceSample` row.

File naming convention: `<FirstName>_<LastName>_embeddings.npy` (case-sensitive,
spaces → underscores). The `Student.embedding_file` column on the DB row must
match this filename exactly.

Embeddings are biometric data. Keep this directory out of version control and
out of any backup that leaves your own infrastructure.
